# Welcome to Contacto Components Library

Storybook helps you build UI components in isolation from our app's business logic, data, and context.
That makes it easy to develop hard-to-reach states. These **stories** are different states of a component which can be independantly developed and tested.

<div className="subheading">Components</div>
The following are the common components that are built. Each of them, has stories, which define their UI states. You can play with it and use it as you wish.

1. [Buttons](?path=/docs/components-button--primary)
2. [Checkbox](?path=/docs/components-checkbox--default)
3. [Radio](?path=/docs/components-radio--default)
4. [Select](?path=/docs/components-select--default)
5. [TextField](?path=/docs/components-textfield--default)
6. [Tag](?path=/docs/components-tag--default)
7. [Card](?path=/story/layout-card--card-with-title)

<div className="subheading">How to use?</div>

1. Install the component package `npm install @contacto-io/style-guide`
2. Import the component that is required - ```import { Button } from '@contacto-io/style-guide'```
3. Visit the particular component page on the storybook app.
4. Navigate to your required story.
5. Copy the code and modify the `props` as required.
6. Since all the components are wrapper components upon antd. Please visit [antd docs](https://ant.design/components/overview/) for additional information.

<div className="subheading">Technical Requirements</div>

#### Dependencies

For the compoenents to work properly, make sure the consumer project has these dependencies installed. The following are the peer dependencies for this component library.

```json
  "peerDependencies": {
    "react": "^16.13.1",
    "react-dom": "^16.13.1",
    "prop-types": "^15.7.2",
    "antd": "^4.16.8",
  }
```

#### CSS variables

This style guide completely depends on CSS variables that are commonly defined by the design team. Make sure that the variables are included on the top for the style guide to work properly.
- [CSS Variable list](?path=/story/colors-css-variable-list--page)

#### Icons

The icons used are taken from the material icon package. So for icons to work properly, make sure material-icon package is included.

```css
@import url('https://fonts.googleapis.com/icon?family=Material+Icons&display=block');
```
